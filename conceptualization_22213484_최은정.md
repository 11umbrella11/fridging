### **1. Business purpose**

<br />

최근 한국 사회는 과거와는 다른 형태의 가구, 1인가구가 증가하며 여러 인프라나 사회 분위기또한 1인 가구를 타겟으로 한 제품이나 가구들이 많아졌다. 또한 배달 문화의 발달로 현대인의 식문화 또한 과거와 많이 달라지며 전 세계의 다양한 음식을 집에서 먹을 수 있게 되었지만, 혼자서 소화하기 힘든 양의 식재료나 잔여 음식물들은 관리가 어려워 번번이 폐기가 되고 있다. 한국농촌경제연구원의 2019 식품소비행태조사에 따르면 1인가구는 전체 음식물쓰레기 비율에서 '먹고 남은 음식'이 48.2%, '상하거나 오래된 음식'이 14.7%로, 다른 가구보다 월등히 높았다.[1]

2019년도보다 현재는 각 시,도 차원에서 음식물 쓰레기 감량 정책을 펼쳐 조금씩 줄여 나가고는 있지만, 아직도 많은 양의 음식물 쓰레기가 폐기되고 있다. 근본적으로 해당문제를 해결하기 위해서는 실제 배출하는 사람들이 가지고 있는 식품의 관리를 쉽게하고, 상하지않도록 도와야 한다. Fridging에서는 사용자가 식품을 구매하거나 배달 음식을 시킨 후 남은 음식을 냉장고에 보관할 때, 해당 음식의 정보를 앱에 입력을 하면 소비기한이 되기 전에 활용가능한 레시피를 안내하거나 냉장고에 보관중인 식품들의 정보를 바탕으로 장보기 리스트를 추천하도록 한다. 이를 통해 사용자가 냉장고를 관리하고 음식을 소비하면서 사용기간 내에 활용할 수 있도록 돕는 것을 목표로 한다. 2025년 10월 배달앱 1위 "배달의 민족" 월간 활성 사용자수가 약 2170만명을 차지하고 1인당 월 평균 10만원을 소비[2]하고 있는 현재, 1인가구나 배달음식을 주 2,3회이상 시켜먹는 사람들을 타켓으로 생각 할 수 있다.

### **3. Use case list**.

<br />

#### 1. 회원가입

|  Actor   |                                            Description                                             |
| :------: | :------------------------------------------------------------------------------------------------: |
| User, DB | 사용자는 서비스를 이용하기위해 어플에 등록한다. 입력된 정보를 DB에 입력하며 신규 사용자로 등록한다 |

<br />

#### 2. 로그인

|  Actor   |                                                                                                   Description                                                                                                   |
| :------: | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------: |
| User, DB | 사용자는 회원가입때 입력했던 아이디,비밀번호를 입력해 어플에 로그인한다. 입력된 로그인정보 (아이디, 비밀번호)를 바탕으로 DB에서 사용자 확인 후 등록 되어있는 경우 1, 아닐 경우 0을 전달해 로그인 여부를 넘긴다. |

<br />

#### 3. 제품 입력

|  Actor   |                                                  Description                                                   |
| :------: | :------------------------------------------------------------------------------------------------------------: |
| User, DB | 사용자가 어플에 입력하고 싶은 제품의 바코드를 인식하거나 직접 제품정보를 입력하면 입력된 정보를 DB에 저장한다. |

<br />

#### 4. 유통기한 경고

|   Actor   |                                                    Description                                                    |

<br />

#### 5. 식품 위치 입력

|  Actor   |                                                Description                                                 |
| :------: | :--------------------------------------------------------------------------------------------------------: |
| User, DB | 입력해둔 식품 이미지를 소유하고있는 냉장고와 같은 구조의 이미지 위치에 배치하면, 위치정보를 DB에 저장한다. |

<br />

#### 6. 장보기 리스트 추천

### **4. Concept of operation**.

<br />

### **1. 회원가입**

| **Purpose**  | 신규 사용자 등록                                                                                                                                                                   |
| :----------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Approach** | 사용자가 가입에 필요한 정보를 입력하면 동일한 정보를 가진 사용자가 있는지 DB에서 확인하여 없을 경우 정보를 DB에 로그인정보(아이디, 비밀번호)를 설정해 사용자 정보와 함께 저장한다. |
| **Dynamics** | 앱 사용을 위해 로그인을 위한 가입이 필요한 경우                                                                                                                                    |
| **Goals**    | 회원가입으로 새로운 이용자의 사용자정보를 받고 로그인 정보를 설정한다.                                                                                                             |

<br />

### **2. 로그인**

| **Purpose**  | 앱사용을 위한 사용자 정보 확인 및 접속 허가                                                                                                                                    |
| :----------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Approach** | 사용자가 로그인정보(아이디, 비밀번호)를 입력하면 가입된 사용자인지 DB에서 검색 후 있을 경우 접속 허가를 하고, 사용자가 해당 앱에서 나갈 경우 접속해제(로그아웃)을 하도록 한다. |
| **Dynamics** | 해당 앱을 이용하기위해 접속할 경우                                                                                                                                             |
| **Goals**    | 로그인 한 사용자에게 접속을 허가한다.                                                                                                                                          |

<br />

### **3. 제품입력**

| **Purpose**  | 보관할 식품정보를 입력                                                                                                                                                       |
| :----------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Approach** | 냉장고에 보관할 식품의 정보를 카메라를 이용해 바코드를 인식하거나 이름을 직접 입력하여 해당 제품에 대한 기본적인 정보를 인터넷에서 가져와 DB에 저장하고 유통기한을 입력한다. |
| **Dynamics** | 냉장고에 보관할 식품정보를 기록할 경우                                                                                                                                       |
| **Goals**    | 냉장고에 음식을 보관하기전에 유통기한을 저장한다.                                                                                                                            |

<br />

### **4. 유통기한 경고**

| **Purpose**  | 소비기한 종료를 경고                                                                                                                                                                                |
| :----------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Approach** | DB에 입력된 음식의 유통기한이 7일보다 적게 남았을 경우 메인 화면 상단에 알림을 띄우고 각 식재료 하단에 각 식품을 활용한 레시피의 링크를 띄워서 사용자가 해당 식품을 소비할 수 있는 방법을 안내한다. |
| **Dynamics** | 냉장고에 보관중인 음식의 유통기한이 임박해 올 경우                                                                                                                                                  |
| **Goals**    | 음식을 소비기한내에 해결하기위한 방법을 제시한다.                                                                                                                                                   |

<br />

### **5. 식품위치 입력**

| **Purpose**  | 냉장고 내부구조를 손쉽게 이미지화                                                                                                                                      |
| :----------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Approach** | 사용자가 입력해둔 식품 이미지를 사용자가 소유하고 있는 냉장고와 같은 구조의 이미지에 보관 중인 위치와 동일하게 옮겨두어 손쉽게 식품의 위치를 찾기위한 지표를 제시한다. |
| **Dynamics** | 냉장고에 보관중인 음식의 위치를 찾을 경우                                                                                                                              |
| **Goals**    | 요리를 할때 필요한 재료를 한번에 찾아서 꺼내기위해 돕는다.                                                                                                             |

<br />

### **6. 장보기 리스트 추천**

| **Purpose**  | 냉장고 재료를 기반으로 장보기 리스트 제공                                                                                                                        |
| :----------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Approach** | 소비기한이 임박한 식재료를 활용하기위한 레시피 중 선택한 레시피의 재료에서 냉장고나 주변에 소유하고 았는 것을 제외하여 장을 보기위한 구매리스트를 화면에 띄운다. |
| **Dynamics** | 식재료를 소비하기위한 레시피를 해보기 위해 부가적으로 필요한 재료를 찾을 경우                                                                                    |
| **Goals**    | 가지고있는 재료를 또 구매하지 않도록 돕는다.                                                                                                                     |

<br />

### **7. 레시피 추천**

| **Purpose**  | 유통기한 임박재료를 활용한 레시피 제공                                                                     |
| :----------- | :--------------------------------------------------------------------------------------------------------- |
| **Approach** | 소비기한이 7일보다 적게 남은 식품을 활용하기위한 레시피를 다른 사용자가 추천한 순서대로 정렬하여 표시한다. |
| **Dynamics** | 소비기한이 적게 남은 음식을 활용하기 위한 레시피를 쉽게 정리하여 보고싶을 경우                             |
| **Goals**    | 사용자가 빠르게 음식을 활용하기위해 레시피를 찾을 시간을 단축시킨다.                                       |

<br />

### **8. 신규 레시피 등록**

| **Purpose**  | 신규 레시피 등록                                                                                |
| :----------- | :---------------------------------------------------------------------------------------------- |
| **Approach** | 관리자가 새로 생겨난 레시피를 DB에 등록하고, 해당 요리를 하는데 필요한 재료들을 같이 등록해둔다 |
| **Dynamics** | 기존에 입력된 레시피 말고 다른 레시피로 요리를 하고싶을때                                       |
| **Goals**    | 사용자가 요리에 질리지않도록 다양한 레시피를 제공한다.                                          |

<br />

### **9. 사용자 의견 전달**

| **Purpose**  | 앱 이용에 대한 사용자 의견 전달                                                                               |
| :----------- | :------------------------------------------------------------------------------------------------------------ |
| **Approach** | 앱을 사용함에 있어서 불편했던 점이나 해결해야할 문제를 기록하여 DB에 저장해두었다가 리스트로 생성해 출력한다. |
| **Dynamics** | 관리자가 앱에서 개선사항을 찾고자할때                                                                         |
| **Goals**    | 관리자가 어플에서 발생한 오류를 빠르게 파악할 수 있도록 돕는다.                                               |

## **5. Problem statement**.

<br />

Fridging는 사용자가 입력하는 음식정보를 바탕으로 소비기한 경고를 하고 빠른 섭취를 위한 레시피 제공 및 장보기 추천기능을 제공하는 만큼, 사용자가 등록한 입력값에 의존하여 여러 작업이 진행이 된다. 앱이 추구하는 목적인 '앱 사용자의 냉장고 사용의 부담감을 덜어주고 소비기한 내 소비를 돕는다'의 목적이 제대로 수행되기 위해서는 몇몇 작업이 전제되어야 한다.

<br />
- Problem (1): 소비기한 확인  
  <br />
  제품의 유통기한이 제대로 입력되어야 한다. 식품류는 제조사나 식품의 분류에따라 유통기한이 입력된 위치나 별도의 계산을 필요로 한다. 표면에 적힌 글자가 있을 수도 있지만 식품성분표 하단에 같이 인쇄되어 있는 경우도 있어서 찾는데 시간이 제법 소요되기도 한다.

  <br />

- Problem (2): 식품위치 배치  
  <br />
  냉장고 위치는 유동적으로 바뀔 수 있다. 사용자가 냉장고에 음식을 보관한 후 앱에서 배치한 위치에 음식사진을 끌어와서 가시적으로 냉장고의 내부구조를 확인 할 수 있게 만든 기능이지만, 사용자가 재료를 사용하거나 용기를 옯겨서 부피가 줄어들게되 효율적인 배치를 위해 자연스럽게 위치를 옮기게 될 수 있다. 이런 경우 앱에서 제공하는 위치에 찾는 음식이 없을 수 도 있다.

- Problem (3): 바코드 인식기능

바코드를 이용할 경우 네트워크 연결을 필수로 한다. 사용자가 직접 식품정보를 입력하기 어려울때 바코드를 인식해 기본정보를 끌어오기 위해서는, 카메라를 통해서 촬영한 바코드의 값을 인터넷으로 넘겨 '유통 데이터 서비스 플랫폼'이나 '코리안넷' 등과 같은 사이트에서 정보를 받아와야하기 때문에 네트워크 연결이 필수적이다. 국내외 식품류의 정보를 모두 DB에 저장하기엔 너무나도 많기 때문이다.
